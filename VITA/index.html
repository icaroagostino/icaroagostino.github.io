API para Otimização Operacional caso VITA

Esse documento descreve a API desenvolvida para Otimização Operacional do caso VITA, o objetivo principal dessa aplicação é fornecer um modelo para alocação da montagem de ventiladores, assim como para a produção das partes e transporte.

A aplicação foi separada em duas partes:

  • Alocação do Centro de Montagem: a escolha do centro de montagem final para os ventilador baseada no tempo total entre a montagem e entrega final.

  • Alocação da Produção das Partes: a escolha da célula de produção final de cada parte do ventilador baseado no tempo total entre a produção de cada parte e a entrega para montagem.

Dessa forma, a aplicação foi implementada utilizando a arquitetura REST em conjunto com o framework swagger.

Essa versão inicial da aplicação está hospedada em uma máquina virtual da AWS e pode ser consultada no link:

  • http://ec2-18-231-56-101.sa-east-1.compute.amazonaws.com:8888/__swagger__/

1. Alocação do Centro de Montagem

O modelo de alocação desenvolvido foi adaptado do projeto FASTEN. A otimização realizada executa um algorítimo de busca exaustiva considerando todas as opções disponíveis dentro de um espaço amostral definido. Além dos dados consumidos pela aplicação via sistema web, foi utilizado para o calculo e estimação dos tempos de transporte a API externa do Open Route Service, sendo uma aplicação open source para geolocalização, permitindo estimar tempos de transporte de forma dinâmica entre múltiplas localizações.

A escolha do centro de montagem final para os ventilador é definida baseada na otimização local do tempo total de fila e transporte entre os centros de montagens disponíveis e o local final de entrega do produto.

Para a implementação da aplicação foram gerados dados randomizados. Na aplicação futura integrada ao sistema de gestão esses dados devem ser consultados em um banco de dados fornecido pelo sistema web.

1.1 Dados de entrada

Para a Alocação do Centro de Montagem foram definidos para essa versão os seguintes inputs:

  • produto: tipo de produto a ser montado no centro.
  • qtd: quantidade o produto a ser montado no centro.
  • lon: longitude do local de entrega do produto final.
  • lat: latitude do local de entrega do produto final.
  • orderID: um identificador para a ordem de montagem, nesse caso deve ser uma string única enviada pelo sistema ou usuário da aplicação. O identificador é retornado como parte do objeto de saída.

Um exemplo de entrada em .json para uma requisição de alocação de montagem será:

{
  "orderID": "2ad1ee0d-ee75-4dab-a27c-e356be6776ee",
  "produto": "ventilador tipo 1",
  "qtd": 4,
  "lon": -46.2903,
  "lat": -23.8647
}

2. Alocação da Produção das Partes

A escolha da célula de produção final de cada parte do ventilador é definida baseada na otimização local do tempo total de produção, fila e transporte entre as células de produção e os centros de montagens disponíveis. Assim como para o caso da montagem, o modelo aplica uma busca exaustiva utilizando dados consultados em banco e a API externa o Open Route Maps.

Para a implementação da aplicação, assim como no caso anterior, foram gerados dados randomizados que na aplicação devem ser consultados em um banco fornecido pelo sistema web.

2.1 Dados de entrada

Para a Alocação da Produção das Partes foram definidos para essa versão os seguintes inputs:

  • tipoServ: tipo de serviço a ser executado (Ex.: impressão 3d, CNC, eletrônica e etc).
  • parte: tipo de produto a ser produzido na célula.
  • qtd: quantidade o produto a ser montado no centro.
  • lon: longitude do local de entrega do produto final.
  • lat: latitude do local de entrega do produto final.
  • orderID: um identificador para a ordem de produção, nesse caso deve ser uma string única enviada pelo sistema ou usuário da aplicação. O identificador é retornado como parte do objeto de saída.

Um exemplo de entrada em .json para uma requisição de alocação de montagem será:

{
  "orderID": "2ad1ee0d-ee75-4dab-a27c-e356be6776ee",
  "tipoServ": "print3d",
  "parte": "pr2",
  "qtd": 3,
  "lon": -46.2903,
  "lat": -23.8647
}
